# Veridra 2.10.0

Veridra 2.10.0 adds explicit outbound SMTP email delivery for captured audit leads and project monitoring summaries.

## Added

- Environment-backed SMTP configuration with STARTTLS or implicit TLS.
- Lead-form notification recipients.
- Project monitoring-summary recipients.
- Plain-text and escaped HTML messages.
- Strict ten-second SMTP timeout and TLS certificate validation.
- Maximum generated message size of 256 KB.
- Persistent delivery attempts with recipient, type, timestamp, status, error, message digest, attempt number and related assessment reference.
- Best-effort lead notifications after lead and assessment persistence.
- Monitoring summaries after successful individual or bounded batch runs.
- Lead and project email-delivery history.
- Explicit manual retry actions.

## Environment contract

The runtime reads `VERIDRA_SMTP_HOST`, `VERIDRA_SMTP_PORT`, `VERIDRA_SMTP_ENCRYPTION`, `VERIDRA_SMTP_SENDER`, `VERIDRA_SMTP_SENDER_NAME`, `VERIDRA_SMTP_USERNAME` and the password variable selected by `VERIDRA_SMTP_PASSWORD_ENV`.

## Secret boundary

SMTP passwords are never stored in lead forms, projects, assessments, reports or delivery records. They are read only from the configured environment variable at delivery time.

## Operational limits

This release does not provide a background queue, automatic retry scheduler, inbound email, marketing automation, CRM-specific adapters, public SaaS secret management or guaranteed delivery while Veridra is not running.
