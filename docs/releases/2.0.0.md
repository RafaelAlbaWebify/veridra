# Veridra 2.0.0

Veridra 2.0.0 adds the first complete local remediation workflow for saved client projects.

## Added

- Deterministic remediation task model and atomic local JSON storage.
- Project-level task lists and status filters.
- Task creation directly from attention findings in the latest saved project assessment.
- Explicit task statuses: open, planned, in progress, fixed, ignored, accepted risk, verification required, and verified.
- Owner labels, due-date text, notes, source-assessment links, edit and delete actions.
- A global task-project landing page at `/tasks`.
- A complete runtime composition entrypoint at `veridra.runtime:app` and the `veridra-api` command.
- Route-level regression tests covering creation, filtering, editing, deletion, escaping, invalid findings and cross-project access.

## Scope

Task state changes remain explicit operator actions. Veridra does not automatically claim that remediation is fixed or verified. Storage remains local to the configured Veridra data directory; this release adds no accounts, cloud sync, scheduling, notifications or multi-user assignment.
