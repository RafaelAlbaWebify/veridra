# Veridra 3.3.0

Veridra 3.3.0 adds a local workspace-membership foundation for seat-aware agency operations.

## Added

- Stable local member records with validated email, display name, role, status and timestamps.
- Owner, administrator, analyst, sales and viewer roles.
- Deterministic capability mappings for workspace, members, projects, assessments, reports, leads, monitoring and tasks.
- Workspace-plan seat enforcement for member creation and reactivation.
- Protection against disabling, demoting or deleting the last active owner.
- Explicit first-owner setup through the local operator workflow.
- Member creation, editing, activation, deactivation and deletion controls.
- Member CSV export.
- Append-only local operator audit events for member and assignment changes.
- Audit-trail dashboard and CSV export.
- Backward-compatible optional member references for project contacts, lead owners and task owners.
- A local assignment dashboard that lists active workspace members and preserves legacy free-text labels.
- Controlled rejection of missing or inactive member selections before commercial records are changed.
- Resolved active-member display with fallback to retained legacy text when a reference is absent, inactive or deleted.

## Compatibility

Existing project, lead and task JSON records remain readable because member-reference fields are optional. Legacy contact and owner labels remain stored and editable. No forced data migration or automatic reassignment is performed.

## Security and identity boundary

These member records are local operational metadata. They do not provide passwords, login sessions, MFA, SSO, identity verification, authenticated request attribution, tenant isolation or public deployment security.

An empty audit actor means the action was performed through the unauthenticated local operator interface. It must not be interpreted as proof of a specific person.

Assignments describe local operational responsibility only. They do not authorize requests or prove that the selected person performed an action.

## Validation

The release is eligible to merge only after exact-head Ruff, strict mypy, pytest, deterministic repository audit and Chromium browser audit succeed.
