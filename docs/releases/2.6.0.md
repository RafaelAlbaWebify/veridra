# Veridra 2.6.0

Veridra 2.6.0 adds a larger deterministic audit batch over the HTML and response headers already collected by the bounded crawler.

## Accessibility readiness

Seven first-class Accessibility findings now cover:

- root HTML language declaration;
- responsive viewport declaration;
- detectable labels for non-hidden form controls;
- detectable names for links and buttons;
- image alt-attribute coverage while preserving explicit empty alt text as valid for decorative images;
- heading-level continuity;
- duplicate non-empty element IDs.

Evidence contains bounded affected-page URLs. These checks are static HTML heuristics. They do not establish WCAG conformance, test rendered states, execute JavaScript or validate behavior with assistive technology.

## Deeper passive security posture

Seven additional passive findings cover:

- observed cookie Secure, HttpOnly and SameSite attributes;
- cross-origin form actions;
- HTTP form actions;
- target=_blank links without detectable noopener or noreferrer isolation;
- absolute HTTP resources;
- Server and X-Powered-By disclosure values;
- unsafe-inline and unsafe-eval directives in an observed Content Security Policy.

The checks use only collected public HTML and headers. They do not submit forms, manipulate cookies, probe ports, exploit vulnerabilities or inspect authenticated behavior.

## Product integration

Accessibility is exposed as an operator assessment area and as a separate no-registration free tool. All new findings automatically flow through existing HTML reports, PDF reports, evidence ZIPs, saved history, comparisons and remediation-task creation.

Collection remains sequential and bounded by the selected crawl profile. Existing public-target, redirect and SSRF protections remain unchanged.
