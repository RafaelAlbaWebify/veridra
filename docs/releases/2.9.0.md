# Veridra 2.9.0

Veridra 2.9.0 adds observable signed webhook delivery for captured audit leads.

## Added

- Optional HTTPS webhook URL per saved lead form.
- Optional HMAC-SHA256 signing secret.
- Deterministic bounded `lead.created` JSON payloads.
- Consent, lead identity, assessment reference, summary and area-summary delivery.
- Public-address validation before each outbound request.
- Five-second timeout, no redirect following and no credential-bearing webhook URLs.
- Persistent delivery-attempt records with attempt number, time, outcome, HTTP status, error and payload digest.
- Delivery history on local lead-detail pages.
- Explicit operator-triggered webhook retry.
- Failure isolation so downstream delivery problems do not discard a successfully captured lead.

## Boundaries

This release does not add email delivery, CRM adapters, background workers, distributed queues, scheduled retries, webhook endpoint authentication beyond optional HMAC signatures, or public deployment hardening.
