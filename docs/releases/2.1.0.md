# Veridra 2.1.0

## Embedded audit lead generation

- Adds reusable local lead-form configurations with organisation labels, headings, introductions, submit labels, consent wording, optional company and phone fields, allowed origins and report-profile references.
- Adds an isolated public audit form at `/embed/audit/{form_id}` that does not expose local projects, profiles, history, tasks or lead-management data.
- Requires website, name, email and explicit consent before running the existing bounded public assessment.
- Stores captured leads and source assessment identifiers atomically in local JSON storage.
- Adds local lead list, detail, status, notes, deletion and CSV export workflows.
- Supports `new`, `contacted`, `qualified`, `won`, `lost` and `deleted_pending` lead states.
- Adds per-form, per-client in-memory rate limiting for loopback evaluation.
- Normalizes allowed origins while omitting default HTTP and HTTPS ports.
- Escapes submitted and configured content and rejects invalid or unknown fields.

## Scope boundaries

This release remains loopback-local. It does not provide authentication, distributed rate limiting, email delivery, webhooks, CRM synchronization, analytics, cloud storage, remote assets or production deployment guarantees.
